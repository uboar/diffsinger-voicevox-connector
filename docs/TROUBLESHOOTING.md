# 困ったときは（トラブルシューティング）

> **対象読者**: 本コネクターを導入したものの、思ったとおりに動かなくて困っているかた向けです。

代表的な症状ごとに、考えられる原因と対処法をまとめています。上から順に試してみてください。

---

## 症状 1: VOICEVOX の歌手リストに DiffSinger が出てこない

「ソング」画面の歌手選択を開いても、DiffSinger 系の歌手が一切表示されない症状です。

### 考えられる原因

- コネクター本体（エンジン）がそもそも追加されていない
- エンジンは追加されたが、`models/` フォルダにモデルが置かれていない
- モデルの配置場所や階層が間違っている

### やってみること

1. **エンジン一覧を確認する**
   VOICEVOX エディタの「設定」→「エンジン」を開き、「DiffSinger Connector」が一覧にあるか確認します。なければ [INSTALL.md](INSTALL.md) を参照して `.vvpp` の追加からやり直してください。
2. **モデルが置かれているか確認する**
   エンジン一覧の「DiffSinger Connector」横の「フォルダを開く」から `models/` フォルダを開き、中にモデルフォルダがあるか確認します。空の場合は [MODEL_SETUP.md](MODEL_SETUP.md) を参照してください。
3. **フォルダ階層を確認する**
   `models/モデル名/dsconfig.yaml` という階層になっているのが正解です。`models/モデル名/モデル名/dsconfig.yaml` のように二重になっていると認識されません。
4. **エディタを再起動する**
   モデルの追加・削除はコネクター起動時に読み込まれます。VOICEVOX エディタを一度完全に終了して、もう一度起動してみてください。

## 症状 2: 「エンジンが起動していません」と表示される

エディタを開くと「DiffSinger Connector エンジンが起動していません」のようなメッセージが出る症状です。

### 考えられる原因

- セキュリティソフトやファイアウォールがコネクターをブロックしている
- 別のソフトが同じポート番号（既定: 50122）を使用している
- コネクターの実行ファイルが破損している、または途中で終了している

### やってみること

1. **少し時間を置いて「エンジンを再起動」する**
   起動には十秒以上かかることがあります。「設定」→「エンジン」から「再起動」ボタンを押してみてください。
2. **セキュリティソフトの確認**
   ウイルス対策ソフトが `DiffSingerConnector.exe` をブロックしていないか確認し、必要であれば「許可」を追加してください。
3. **ポートの競合を確認する**
   既定ではポート 50122 を使います。他のソフトが同じポートを使っていると起動できません。PC を再起動すると競合が解消されることが多いです。
4. **インストールし直す**
   GitHub Releases から最新の `.vvpp` を再ダウンロードし、もう一度ダブルクリックで上書きインストールしてみてください。

## 症状 3: 歌わせても無音、またはノイズしか出ない

歌手は選べて再生もできるが、音声がおかしい症状です。

### 考えられる原因

- モデルが日本語に対応していない（中国語専用モデルなど）
- `phonemes.txt` の内容がコネクターと合っていない
- 歌詞がひらがな以外（漢字・カタカナ・英字）で書かれている
- ノートの長さが極端に短い／長い

### やってみること

1. **歌詞をひらがなに統一する**
   現時点では「ひらがな」のみが対象です。「カタカナ」「漢字」「英語」は読み上げられません。
2. **モデルが日本語対応か確認する**
   配布元のページで「日本語」「Japanese」「JA」と明記されているモデルか確認します。
3. **PC のスピーカー音量・ミュートを確認する**
   念のため OS の音量設定と、VOICEVOX エディタ右下の音量ミキサー的な部分も確認してください。
4. **別のモデルで試す**
   同じ症状が複数モデルで出る場合は、コネクター側の不具合の可能性があります。GitHub Issues にご報告ください。

## 症状 4: 起動が遅い／モデル読み込みエラーが出る

VOICEVOX 起動後、しばらくしてもエンジンが「準備中」のままだったり、エラーになる症状です。

### 考えられる原因

- モデルファイルが大きく、初回読み込みに時間がかかっている（特に SSD 以外の場合）
- モデルファイルが破損している、ダウンロードが途中で失敗している
- メモリ（RAM）が不足している

### やってみること

1. **数分待ってみる**
   初回起動は数十秒〜数分かかる場合があります。特に大きなモデルや複数モデルを置いている場合は時間がかかります。
2. **モデルを 1 つだけにして試す**
   `models/` の中身を一度別の場所に退避し、確実に動くモデル 1 つだけにして試してください。原因のモデルを特定できます。
3. **ファイルサイズを確認する**
   配布元に書かれているファイルサイズと、実際にダウンロードできたファイルのサイズが一致しているか確認します。途中で失敗していたら再ダウンロードしてください。
4. **PC を再起動する**
   メモリ不足や前回終了が不完全だった場合に効くことがあります。

## 症状 5: 「CUDA」関連のエラーが出る

「CUDA Provider が見つかりません」「CUDA out of memory」などのメッセージが表示される症状です。

### 考えられる原因

- お使いの PC に NVIDIA 製 GPU がない、または CUDA ドライバーが入っていない
- GPU メモリが不足している（小さい GPU で大きいモデルを動かそうとしている）

### やってみること

1. **CPU モードで動かす**
   本コネクターは GPU が無くても CPU で動作します。CUDA エラーが出ても、自動的に CPU にフォールバックして動き続ける設計です。生成は遅くなりますが、ひとまず動作するはずです。
2. **エラーが出るが動いている場合は無視して構いません**
   「CUDA Provider 利用不可、CPU で続行します」のようなメッセージは想定内の動作です。
3. **GPU を使いたい場合**
   NVIDIA 公式サイトから最新のグラフィックドライバーをインストールしてみてください。GPU の VRAM が 4GB 未満の場合は、GPU を使わない設定をおすすめします。

## それでも解決しないとき

### ログファイルを確認する

コネクターは動作ログを以下のファイルに出力しています。

- **場所**: 本コネクターのインストール先フォルダ内の **`logs/connector.log`**
- **開き方**: 「設定」→「エンジン」→「DiffSinger Connector」横の「フォルダを開く」で開いたフォルダの中の `logs` フォルダ

このファイルを「メモ帳」などのテキストエディタで開くと、エラーの詳細が日本語で記録されています。

### GitHub Issues で報告する

それでも解決しない場合は、GitHub のリポジトリの **Issues** タブから新しい issue を作成してご報告ください。以下のテンプレートをコピーしてご利用ください。

````markdown
## 環境
- OS: （例: Windows 11 / macOS 14.5）
- VOICEVOX バージョン: （例: 0.20.0）
- DiffSinger Connector バージョン: （例: 0.1.0）
- 使っているモデル名と配布元: （差し支えなければ）

## 症状
（どんな操作をしたら、どうなったか。可能であればスクリーンショット）

## 期待した動作
（どうなってほしかったか）

## ログ
`logs/connector.log` の関連部分を貼り付けてください（個人情報が含まれていないことをご確認ください）。

```
ここにログを貼る
```
````

> **注意**: ログには PC 上のフォルダパスが含まれることがあります。個人名などが含まれている場合は伏せ字にしてからご報告ください。
